# STO-203镜像加速器是部署前提

> 类型：架构决策 | 项目：_global | 风险：L4 | 状态：草稿
> 同步自 Feishu Bitable 表 A | 2026-05-02T11:06:02Z

## 摘要

镜像加速器配置是部署的硬前提，不是性能优化项。新服务器上线第一件事就是配 daemon.json + restart docker，否则 docker build 必失败。daemon.json 同时把日志轮转一起配进去（见容器健康告警卡）。

## 正文

> 记录日期：2026-04-20  风险等级：L4

## 背景
STO-203 复盘：生产服务器在中国大陆，到 registry-1.docker.io（Docker Hub CDN，Meta 网段）本身不稳。未配置镜像加速器时 docker compose build 会卡在 load metadata for docker.io/library/xxx 阶段超时失败，所有 deploy 因此失败，且现象隐蔽（看起来像代码问题）。

## 内容
必须配置 /etc/docker/daemon.json：{ "registry-mirrors": [ "https://registry.cn-hangzhou.aliyuncs.com", "https://docker.m.daocloud.io" ]}。改完执行 systemctl restart docker（中断所有容器 20-30 秒，选低峰）。验证命令：docker info | grep -A3 'Registry Mirrors'，看到阿里云/DaoCloud 列出 = 配置生效。 当前状态（2026-04-20，基础设施清单 §15）：Store Docker daemon 仍连不上 Docker Hub，标记 P0，阻塞所有 docker build / deploy。 配套：deploy.sh 是无条件 rebuild（只比较 git commit），即便纯文档 PR 也会触发 build → 对镜像加速器依赖很高。

## 结论
镜像加速器配置是部署的硬前提，不是性能优化项。新服务器上线第一件事就是配 daemon.json + restart docker，否则 docker build 必失败。daemon.json 同时把日志轮转一起配进去（见容器健康告警卡）。

## 影响
所有 docker compose build / deploy.sh / CI 构建；新服务器上线流程；docker daemon 配置变更窗口；服务器迁移/重装时 daemon.json 必须先恢复；纯文档 PR 部署也会受影响。

## 下次注意
新服务器上线 SOP 写死“先配镜像加速器 + 日志轮转 + restart docker，再做任何 docker build”；定期 docker info 检查镜像加速器仍生效；服务器迁移/重装第一步恢复 daemon.json；改 daemon.json 选低峰窗口（中断 20-30 秒）。
