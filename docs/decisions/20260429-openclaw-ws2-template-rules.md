## 背景

Workspace 49 文件（WS-1）完成后，第三方 GPT-5.5 审计发现 5 条真问题：断链引用、路径硬编码、不安全 JSON 写入、命令稳健性不足、规则自相矛盾。WS-2 Sprint 落地 6 处规则补丁 + 1 新建文件，目标是「未来所有 Sprint 自动复用，不再返工」。

## 补丁内容

| 编号 | 内容 | 解决的问题 |
|---|---|---|
| P1 | secret 扫描改为扫 PR 内 REPO_ROOT | 原逻辑扫家目录，漏扫 PR 内容 |
| P2 | git rev-parse + ~ 路径变量替代硬编码 | 跨机部署路径不一致 |
| P3 | JSON 写入加 atomic write + setdefault + backup | 原逻辑直接覆写，无备份 |
| P4 | Sprint Fallback 策略 | 无降级预案时 Sprint 中途卡死 |
| P5 | 禁止操作清单明确化 | 隐含禁止不如显式禁止 |
| P6 | Sprint 中间检查点 25%/50%/100% | 无检查点导致偏离无人发现 |
| 新建 | workspace 操作 SOP + Decisions 写作规则 | 操作不规范导致反复修改 |

## 经验教训

- 一次性建 49 文件后必须做第三方审计（不同模型，不同视角）。GPT-5.5 审计 WS-1 的 5 条问题，都是 Claude AI 自己审查时不会发现的盲区。
- Sprint 中间检查点（25%/50%/100%）能在偏离发生时早期发现，而不是等 Sprint 结束才发现方向错了。
- 模板规则本身是一次性投资：WS-2 规则落地后，未来每个 Sprint 自动复用，成本递减。
