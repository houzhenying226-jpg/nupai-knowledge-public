## 正文

> 记录日期：2026-04-17  风险等级：L3

## 背景
两台服务器容器多（Store 21+ / CRM 9+），不配健康告警 + 日志轮转会导致挂了不知道、磁盘被日志撑爆。基础设施清单 §17 整理。

## 内容
容器健康告警：脚本 deploy/health-check.sh；频率每 5 分钟（cron）；告警渠道飞书 webhook（域名 open.larksuite.com）；检测项：容器 NOT RUNNING / RESTARTING；日志 backups/health-check.log。 日志轮转：配置文件 /etc/docker/daemon.json；策略：每个容器最多 3 个日志文件，每个最大 50MB；修改后须 systemctl restart docker 生效（会中断所有容器 20-30 秒，选低峰）；daemon.json 同时还配镜像加速器（registry-mirrors，见 STO-203 卡）。 .env 保护：deploy.sh 部署前自动备份 .env 到 /tmp/.env.deploy.bak；每日异地同步 .env 到 123.57.224.35（职业装1）；.env 修改后必须 recreate 不是 restart 才会重载。

## 结论
新服务器上线必须把 health-check.sh + daemon.json 一起配好。改 daemon.json 选低峰窗口（systemctl restart docker 中断 20-30 秒）。改 .env 一律 recreate。

## 影响
容器异常发现时延（5 分钟级）、磁盘空间消耗、Docker 重启窗口选择、灾备恢复链路、镜像加速器配置（同 daemon.json）。

## 下次注意
