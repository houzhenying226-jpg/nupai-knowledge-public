## 正文

> 记录日期：2026-04-17  风险等级：L4

## 背景
NuPai 涉及 Store 生产、CRM 生产、新加坡 Mac Mini 开发机三台主力机器，外加异地备份机"职业装1"，IP 和登录方式不同，每次 SSH 现查易记错。

## 内容
Store 生产 → ssh root@39.96.216.33（公网），内网 172.25.35.85，阿里云。 CRM 生产 → ssh root@39.106.83.79（公网），内网 172.26.239.156，阿里云。 Mac Mini 开发 → ssh nupai（仅 Tailscale 100.69.20.113 可达），自有机器。 异地备份"职业装1" → 123.57.224.35（接收 Store/CRM 每日备份 + .env 同步），fail2ban SSH 5 次失败封 1 小时。 密码不在任何文档/git/记忆/聊天里，需要时当面问振英。

## 结论
SSH 直接复制以上命令登录。Mac Mini 只能从 Tailscale 网络登录，公网无法访问。三台密码靠面对面交接，不上 IM/邮件/文档。

## 影响
所有 SSH 运维、deploy.sh 部署、备份异地同步、Claude Code 远程开发跳板、Dify（Store 80/443）和 Skyvern（CRM 6080/8000/9222）访问位置。

## 下次注意
SSH 前查本卡确认 IP/用户；密码绝对不写进文档/git/聊天；新机器上线先建本卡再投运维；服务器 IP 变更当天更新基础设施清单 §1。
